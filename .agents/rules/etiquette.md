# testoise Agent Rules

When acting as an agent in this repository, you MUST follow these rules:

1. **Ask Before Committing**: NEVER perform a `git commit` or `git push` without asking for explicit permission from the user first. This is a non-negotiable rule.
2. **Write Specs**: Always write or update tests (specs) for any logic changes.
3. **Lazy Var Patterns**: Follow the patterns established in `skills/testoise.md` (no globals, lazy dependency injection).
4. **🐢 Branding**: Maintain the tortoise-themed branding in documentation and messages.
